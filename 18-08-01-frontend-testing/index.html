A test consists of a precondition, some input, and some expected result.
- A program always consists of a bunch of lines of code. - As complex as everything is it always boils down to: - You have some code, that takes an input, and some output. - A test for this thing now has three components: Precondition, Input, Result - When testing your job is to find inputs that reveal bugs, so unexpected results - Really try to break it, because if you tried that and it didn't you know its probably going to hold up - For example, if you were to call this with say 3 and 0, you would get infinity, which is dumb, but that's javascript for you
